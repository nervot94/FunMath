# User stories

## User Story 1 - Levenssysteem
**Als** speler
**wil ik** een beperkt aantal levens hebben
**zodat** het spel uitdagender wordt en ik niet onbeperkt fouten kan maken

### Acceptatiecriteria
- De speler start met **3 levens**
- Bij een fout antwoord gaat er **1 leven af**
- Het aantal resterende levens wordt zichtbaar weergegeven
- Als het aantal levens **0** is, stopt het spel
- Na game over kunnen geen nieuwe antwoorden meer worden ingevoerd

## User Story 2 - Score bijhouden
**Als** speler
**wil ik** dat mijn score wordt bijgehouden
**zodat** ik kan zien hoe goed ik het doe

### Acceptatiecriteria
- De score start op **0**
- Bij elk goed antwoord wordt de score met **1 verhoogd**
- De score wordt zichtbaar weergegeven op het scherm
- De score **verandert niet** bij een fout antwoord
